What you have now is basically a “mini-game definition card” — good start, but it’s too static. If you want admins to actually create flexible games inside QuestHub, you need to upgrade it into a Mini-Game System Builder, not just a text form.
Right now your problem is:
 👉 everything is description-based
 👉 nothing is enforceable or dynamic
 👉 no structure for automation or validation
So here’s a better way to think about it.
🧠 1. Core Idea: Mini-Game = “Rule Engine Card”
Instead of just:
Name + Description + Rules
You turn every mini-game into:
A structured game engine definition the app can execute
So every game becomes:
WHAT to do
HOW to check success
HOW to reward
HOW to limit abuse
